# Phase 2 — 動作庫頁

> 前置：Phase 1 的 repository 已可用。

## 目標
提供內建動作庫，支援肌群篩選、關鍵字搜尋，以及自訂動作的新增/編輯/刪除。

## 規格
1. 在 `src/data/seed-exercises.ts` 準備 **約 40–60 個常見動作**，依肌群分布（胸/背/腿/肩/二頭/三頭/核心/臀 各 5–8 個），每筆標 `muscleGroup` 與 `equipment`、`isCustom:false`。
   - 接到 Phase 1 的 `seedExercisesIfEmpty()`，首次啟動寫入。
   - （可請 Claude 另外幫你產這份清單。）
2. 動作庫頁（`/exercises`）：
   - 頂部搜尋框（即時關鍵字過濾 `name`）。
   - 肌群快速篩選（chips / 下拉），可「全部」。
   - 動作列表：顯示名稱、肌群、器材標籤。
   - 點動作可看細節（含 notes）。
3. 自訂動作 CRUD：
   - 「新增動作」表單：名稱、肌群、器材、備註 → 存成 `isCustom:true`。
   - 自訂動作可編輯/刪除；**內建動作不可刪除**（可隱藏或不提供刪除）。
4. 全程透過 repository，不直接碰 Dexie。

## 驗收標準
- 搜尋「臥推」找得到相關動作。
- 切換肌群只顯示該肌群動作。
- 新增一個自訂動作 → 重開 App 仍在、可編輯可刪除。

## 注意
- 動作清單之後會被 Phase 3「加入動作」與 Phase 5「選動作看進度」共用，列表元件盡量做成可複用。
