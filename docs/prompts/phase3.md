# Phase 3 — 訓練紀錄核心（App 的心臟）

> 前置：Phase 1 的 activeWorkout store + 即時自動存、Phase 2 的動作選擇可用。

## 目標
完整的「開始訓練 → 逐組紀錄 → 組間休息計時 → 結束訓練」流程，全程即時自動存、可中途離開恢復。

## 規格
1. 訓練頁（`/`）狀態分兩種：
   - **無進行中訓練**：顯示「開始訓練」按鈕（可選填標題如「推日」）。按下建立 `status:'active'` 的 Workout 寫入 store。
   - **有進行中訓練**：顯示本次訓練畫面。
2. 加入動作：從動作庫挑一或多個動作 → 各自成為一張 `WorkoutEntry` 卡（依加入順序 `order`）。
3. 每張動作卡：
   - 逐組顯示 `SetLog` 列（組序、重量、次數、完成勾選）。
   - 輸入用**大數字 + 加減步進器（NumberStepper 元件）**，單手好按；重量步進預設 2.5（kg），次數步進 1。
   - 操作：**加一組**、**刪一組**、**複製上一組的數值**、標記**暖身組**（`isWarmup`）。
   - 可選填 RPE。
4. 休息計時器（RestTimer 元件）：
   - **打勾完成一組時自動啟動倒數**，秒數取該動作 `defaultRestSeconds` ?? 設定的 `defaultRestSeconds`。
   - **用「結束目標時間戳」計算剩餘秒數**（`target = Date.now() + sec*1000`，render 時用 `target − Date.now()`），不可用 setInterval 累加 → 確保鎖屏/切背景回來仍正確。
   - 倒數可 +15s / −15s / 跳過。
   - 結束時依設定 **震動（navigator.vibrate）+ 提示音**。
5. 結束訓練：按「結束訓練」→ 設 `endedAt = Date.now()`、`status:'completed'`、呼叫 `completeWorkout()`、清空 active store。
6. **即時自動存**：以上任何變更（加動作/加組/改值/打勾/暖身/結束）都立即寫回 IndexedDB（沿用 Phase 1 機制）。

## 驗收標準
- 完整紀錄一次訓練：3 個動作、每個 3 組、含 1 個暖身組。
- 打勾某組 → 休息倒數啟動、結束會震動/響。
- 計時中切到別的 App 約 30 秒再回來 → 剩餘秒數正確（沒停、沒亂跳）。
- 紀錄到一半關掉分頁重開 → 接續同一筆進行中訓練。

## 注意
- 這頁互動最密集，元件（SetRow / NumberStepper / RestTimer）拆乾淨、好複用。
- 「複製上一組」要連暖身狀態一起判斷合理預設（通常複製成正式組）。
